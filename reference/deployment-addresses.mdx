## Overview

The DeployableVM contract is deployed using deterministic deployment across multiple EVM-compatible chains. The addresses listed below are the official deployments maintained by the CoW Protocol team.

<Note>
  All deployments use the same address across chains where possible, ensuring consistent integration.
</Note>

## Using the Addresses

To interact with DeployableVM on any supported chain:

### Solidity

```solidity
interface IDeployableVM {
    function execute(
        bytes32[] calldata commands,
        bytes[] memory state
    ) external payable returns (bytes[] memory);
}

// Replace with the address for your target chain
address constant DEPLOYABLE_VM = 0x9585c3062Df1C247d5E373Cfca9167F7dC2b5963;

function executeWeiroll(
    bytes32[] memory commands,
    bytes[] memory state
) internal returns (bytes[] memory) {
    (bool success, bytes memory result) = DEPLOYABLE_VM.delegatecall(
        abi.encodeCall(IDeployableVM.execute, (commands, state))
    );
    require(success, "Weiroll execution failed");
    return abi.decode(result, (bytes[]));
}
```

### TypeScript / ethers.js

```typescript
import { ethers } from 'ethers';

const DEPLOYABLE_VM_ADDRESS = '0x9585c3062Df1C247d5E373Cfca9167F7dC2b5963';

const deployableVM = new ethers.Contract(
  DEPLOYABLE_VM_ADDRESS,
  ['function execute(bytes32[] calldata commands, bytes[] memory state) external payable returns (bytes[] memory)'],
  signer
);

// Note: You must call this via delegatecall from your contract
```

### Foundry

In your Foundry project:

```solidity
import {DeployableVM} from "weiroll/DeployableVM.sol";

contract MyContract {
    DeployableVM constant vm = DeployableVM(0x9585c3062Df1C247d5E373Cfca9167F7dC2b5963);
    
    function myFunction() external {
        bytes32[] memory commands;
        bytes[] memory state;
        
        // ... prepare commands and state
        
        (bool success, bytes memory result) = address(vm).delegatecall(
            abi.encodeCall(vm.execute, (commands, state))
        );
    }
}
```

## Verification

All deployments are verified on their respective block explorers. You can view the source code and verification status by visiting the transaction hash link or searching for the contract address on the chain's block explorer.

## Networks JSON

The deployment addresses are maintained in the [`networks.json`](https://github.com/cowprotocol/weiroll/blob/main/networks.json) file in the repository. This file is programmatically generated from deployment artifacts.

<Tip>
  Import this file directly in your application to automatically use the correct address for each chain.
</Tip>

## Adding New Chains

To request deployment on a new chain, please [open an issue](https://github.com/cowprotocol/weiroll) on the GitHub repository.

## Deterministic Deployment

All contracts are deployed using Foundry's deterministic deployment feature with specific toolchain version:

```bash
foundryup -C cb9dfae298fe0b5a5cdef2536955f50b8c7f0bf5
```

This ensures consistent addresses across chains where possible. See the [deployment guide](/deployment/deterministic-deployment) for more details.
