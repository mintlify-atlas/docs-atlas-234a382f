## What is Weiroll?

Weiroll is a flexible execution system designed to enable trust-minimized and gas-efficient scripting on Ethereum. This deployable implementation provides a virtual machine (VM) that executes on-chain scripts composed of commands and state, enabling complex, multi-step operations to be executed atomically in a single transaction with minimal gas overhead.

This repository wraps the audited [Enso Finance Weiroll VM](https://github.com/enso-finance/enso-weiroll) in a deployable contract that can be safely used by any protocol through delegatecall.

## Why Weiroll?

Traditional smart contract interactions often require multiple separate transactions to complete complex operations, leading to:

- **High gas costs** from multiple transaction overheads
- **Security risks** from intermediate states being exposed
- **Poor user experience** requiring multiple wallet interactions
- **Atomic execution challenges** where partial failures leave the system in an inconsistent state

Weiroll solves these problems by providing a VM that can execute multiple operations atomically within a single transaction, all while maintaining security through its delegatecall-only design.

## How It Works

The DeployableVM contract is a thin wrapper around the Weiroll VM that enforces security constraints:

```solidity
contract DeployableVM is WeirollVM {
    error CallableOnlyViaDelegateCall();

    address private immutable WEIROLL_SINGLETON;

    constructor() {
        WEIROLL_SINGLETON = address(this);
    }

    function execute(bytes32[] calldata commands, bytes[] memory state) 
        public payable returns (bytes[] memory) 
    {
        require(address(this) != WEIROLL_SINGLETON, CallableOnlyViaDelegateCall());
        return _execute(commands, state);
    }
}
```

The immutable `WEIROLL_SINGLETON` address is set at deployment time to the contract's own address. When `execute()` is called:

- **Via delegatecall**: `address(this)` is the caller's address, so the check passes ✓
- **Direct call**: `address(this)` equals `WEIROLL_SINGLETON`, so the check fails ✗

This ensures that Weiroll can only be used in the context of another contract's storage, never operating on its own state.
