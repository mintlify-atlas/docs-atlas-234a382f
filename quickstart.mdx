Get up and running with Weiroll in your smart contract project. This guide shows you how to integrate the DeployableVM into your contracts.

## Installation

<Steps>
  <Step title="Add Weiroll to your project">
    You can reference the deployed DeployableVM contract directly or add it as a dependency to your Foundry project.

    <CodeGroup>
    ```bash Foundry
    forge install cowdao-grants/weiroll
    ```

    ```solidity Direct Reference
    // Just use the deployed address on your target chain
    address constant DEPLOYABLE_VM = 0x9585c3062Df1C247d5E373Cfca9167F7dC2b5963;
    ```
    </CodeGroup>
  </Step>

  <Step title="Create your contract">
    Create a contract that will use Weiroll via delegatecall:

    ```solidity MyContract.sol
    // SPDX-License-Identifier: MIT
    pragma solidity ^0.8.0;

    interface IDeployableVM {
        function execute(
            bytes32[] calldata commands,
            bytes[] memory state
        ) external payable returns (bytes[] memory);
    }

    contract MyContract {
        // DeployableVM address (same on all chains)
        address constant WEIROLL_VM = 0x9585c3062Df1C247d5E373Cfca9167F7dC2b5963;

        function executeScript(
            bytes32[] memory commands,
            bytes[] memory state
        ) external returns (bytes[] memory) {
            // MUST use delegatecall
            (bool success, bytes memory result) = WEIROLL_VM.delegatecall(
                abi.encodeCall(IDeployableVM.execute, (commands, state))
            );
            
            require(success, "Weiroll execution failed");
            return abi.decode(result, (bytes[]));
        }
    }
    ```
  </Step>

  <Step title="Test your integration">
    Create a test to verify the integration works:

    ```solidity MyContract.t.sol
    // SPDX-License-Identifier: MIT
    pragma solidity ^0.8.0;

    import "forge-std/Test.sol";
    import {MyContract} from "../src/MyContract.sol";

    contract MyContractTest is Test {
        MyContract public myContract;

        function setUp() public {
            myContract = new MyContract();
        }

        function testExecuteEmptyScript() public {
            // Empty commands and state for testing
            bytes32[] memory commands = new bytes32[](0);
            bytes[] memory state = new bytes[](0);

            bytes[] memory result = myContract.executeScript(commands, state);
            assertEq(result.length, 0);
        }
    }
    ```

    Run your tests:

    ```bash
    forge test
    ```
  </Step>
</Steps>

## Understanding the Basics

<AccordionGroup>
  <Accordion title="Why delegatecall?">
    DeployableVM **requires** delegatecall for security reasons:

    - It prevents the VM contract itself from being executed directly
    - The script runs in **your contract's context**, accessing your storage and balance
    - This is the intended design pattern for safe VM usage

    Direct calls will revert with `CallableOnlyViaDelegateCall()` error.
  </Accordion>

  <Accordion title="What are commands and state?">
    **Commands**: An array of `bytes32` values, where each command encodes:
    - Target contract address to call
    - Function selector
    - Flags for call type (call/delegatecall/staticcall)
    - Input/output indices into the state array

    **State**: An array of `bytes` values containing:
    - Input parameters for commands
    - Intermediate values between commands
    - Return values from executed commands

    You'll typically use the [Weiroll planner](https://github.com/weiroll/weiroll) to build these arrays.
  </Accordion>

  <Accordion title="Gas considerations">
    The DeployableVM is highly optimized:
    - Compiled with `via_ir = true`
    - Optimizer runs set to 1,000,000
    - Minimal overhead beyond the actual operations being executed

    The gas cost is primarily determined by the operations in your commands, not the VM itself.
  </Accordion>
</AccordionGroup>

## Real-World Example

Here's a complete example from the test suite showing proper usage:

```solidity DeployableVM.t.sol lines
import {DeployableVM} from "../src/DeployableVM.sol";
import {Test, console} from "forge-std/Test.sol";

contract DeployableVMTest is Test {
    DeployableVM public weiVm;

    function setUp() public {
        weiVm = new DeployableVM();
    }

    function test_reverts_if_not_delegatecall() public {
        bytes32[] memory commands = new bytes32[](0);
        bytes[] memory state = new bytes[](0);

        // ❌ This will revert because it's a direct call
        vm.expectRevert(DeployableVM.CallableOnlyViaDelegateCall.selector);
        weiVm.execute(commands, state);
    }

    function test_callable_via_delegatecall() public {
        bytes32[] memory commands = new bytes32[](0);
        bytes[] memory state = new bytes[](0);

        // ✅ This succeeds because it uses delegatecall
        (bool success,) = address(weiVm).delegatecall(
            abi.encodeCall(DeployableVM.execute, (commands, state))
        );
        assertTrue(success);
    }
}
```

<Note>
  See [~/workspace/source/test/DeployableVM.t.sol](https://github.com/cowdao-grants/weiroll/blob/main/test/DeployableVM.t.sol) for the complete test file.
</Note>
